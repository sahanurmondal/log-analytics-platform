## üéØ Key Problem-Solving Patterns

### 1. **Two Pointers Technique**
```java
// Template for palindrome checking
while (left < right) {
    if (chars[left] != chars[right]) return false;
    left++; right--;
}
```
**Use Cases**: Palindromes, sorted array problems, cycle detection
**Problems**: [Valid Palindrome](twopointers/easy/ValidPalindrome.java), [3Sum](twopointers/medium/ThreeSum.java), [Trapping Rain Water](twopointers/hard/TrappingRainWater.java)

### 2. **Sliding Window**
```java
// Variable window template
int left = 0, right = 0;
while (right < arr.length) {
    // Expand window
    window.add(arr[right++]);
    
    // Contract window if needed
    while (windowNeedsContraction()) {
        window.remove(arr[left++]);
    }
}
```
**Use Cases**: Substring problems, subarray optimization
**Problems**: [Longest Substring Without Repeating Characters](slidingwindow/medium/LongestSubstringWithoutRepeatingCharacters.java), [Minimum Window Substring](slidingwindow/medium/MinimumWindowSubstring.java)

### 3. **Fast & Slow Pointers (Floyd's Cycle Detection)**
```java
// Cycle detection template
ListNode slow = head, fast = head;
while (fast != null && fast.next != null) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow == fast) return true; // Cycle detected
}
```
**Use Cases**: Cycle detection, finding middle element
**Problems**: [Linked List Cycle](linkedlist/medium/LinkedListCycle.java), [Find Middle of Linked List](linkedlist/easy/MiddleOfLinkedList.java)

### 4. **Merge Intervals**
```java
// Interval merging template
Arrays.sort(intervals, (a, b) -> a[0] - b[0]);
for (int[] interval : intervals) {
    if (result.isEmpty() || result.get(result.size()-1)[1] < interval[0]) {
        result.add(interval);
    } else {
        result.get(result.size()-1)[1] = Math.max(result.get(result.size()-1)[1], interval[1]);
    }
}
```
**Use Cases**: Overlapping intervals, scheduling problems
**Problems**: [Merge Intervals](intervals/medium/MergeIntervals.java), [Meeting Rooms II](intervals/hard/MeetingRoomsII.java)

### 5. **Tree BFS/DFS**
```java
// Tree DFS template
void dfs(TreeNode node) {
    if (node == null) return;
    // Process current node
    dfs(node.left);
    dfs(node.right);
}

// Tree BFS template
Queue<TreeNode> queue = new LinkedList<>();
queue.offer(root);
while (!queue.isEmpty()) {
    TreeNode node = queue.poll();
    // Process current level
    if (node.left != null) queue.offer(node.left);
    if (node.right != null) queue.offer(node.right);
}
```
**Use Cases**: Tree traversal, level processing
**Problems**: [Binary Tree Level Order Traversal](trees/medium/BinaryTreeLevelOrderTraversal.java), [Maximum Depth](trees/easy/MaximumDepthOfBinaryTree.java)

### 6. **Backtracking**
```java
// Backtracking template
void backtrack(List<Integer> current, int start) {
    result.add(new ArrayList<>(current));
    
    for (int i = start; i < nums.length; i++) {
        current.add(nums[i]);
        backtrack(current, i + 1);
        current.remove(current.size() - 1); // backtrack
    }
}
```
**Use Cases**: Permutations, combinations, constraint satisfaction

// 1D DP template
int[] dp = new int[n + 1];

    dp[i] = Math.max(dp[i-1], dp[i-2] + nums[i]); // Example transition
}

int[][] dp = new int[m + 1][n + 1];
for (int i = 1; i <= m; i++) {
    for (int j = 1; j <= n; j++) {

    }
}

## üîß Performance Optimization Tips

### **Time Complexity Optimization**
- Use **Hash Maps** for O(1) lookups instead of O(n) linear search
- Apply **Binary Search** on sorted data for O(log n) search
- Use **Two Pointers** instead of nested loops for O(n) solutions
- Implement **Sliding Window** for subarray/substring problems

### **Space Complexity Optimization**  
- **In-place algorithms** when possible (modify input array)
- **Iterative vs Recursive** to avoid call stack overhead
- **Bit manipulation** for space-efficient boolean arrays
- **Rolling arrays** in DP to reduce space from O(n) to O(1)
